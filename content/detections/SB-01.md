<h1>SB-01 Detection Map</h1>

<p class="summary">
  이 페이지는 2024 PipelineKey의 Techniques Used를 기준으로 어떤 로그를 수집하고, 각 로그의 어떤 필드로 공격 행위를 확인할 수 있는지 정리하는 초안이다.
  Campaign Page 1이 공격 흐름과 Technique 매핑을 설명한다면, 이 Page 2는 Technique별 로그 근거와 Investigation Pivot을 정리한다.
</p>

<div class="concept-banner">
  <b>문서화 원칙</b>
  <span>Technique 문서와 Log 문서를 분리한다. Technique은 관련 Log ID를 참조하고, Log Catalog는 해당 로그가 커버하는 Technique을 역매핑한다.</span>
</div>

<h2 id="detection-details">Detection Details</h2>

<details class="technique-panel" open>
  <summary><span class="id-cell">T1021.004</span> Remote Services: SSH</summary>
  <div class="technique-body">
    <p>Jenkins에서 확보한 배포 키와 deploy 계정을 이용해 App 서버에 SSH 접속하는 단계다. 이 Technique은 App 서버의 인증 로그를 1차 근거로 보고, Jenkins 배포 시간대와 교차 확인한다.</p>
    <div class="mapping-grid">
      <div>
        <h3>Log IDs</h3>
        <p><a href="../logs/LL-001.html">LL-001 Linux SSH Auth Log</a></p>
      </div>
      <div>
        <h3>Representative Path</h3>
        <p><code>/var/log/auth.log</code></p>
      </div>
      <div>
        <h3>Fields to Read</h3>
        <p><code>@timestamp</code>, <code>user.name</code>, <code>source.ip</code>, <code>event.outcome</code>, <code>process.name</code></p>
      </div>
    </div>
    <h3>Detection Point</h3>
    <ul>
      <li>App 서버에서 <code>Accepted publickey for deploy</code> 이벤트 확인</li>
      <li><code>source.ip</code>가 Jenkins 서버인지, 또는 비정상 출발지인지 확인</li>
      <li>Jenkins 정상 배포 시간대와 SSH 접속 시간이 일치하는지 비교</li>
    </ul>
    <h3>KQL Draft</h3>
    <pre><code>sb01_server: "app" and sb01_log_type: "linux_auth" and message: "Accepted publickey for deploy"</code></pre>
    <h3>Investigation Pivot</h3>
    <p><code>source.ip</code> -> Jenkins 배포 이력 -> App 서버 auditd 경로 접근 -> DB 접속 로그 순으로 확장한다.</p>
  </div>
</details>

<details class="technique-panel">
  <summary><span class="id-cell">T1083</span> File and Directory Discovery</summary>
  <div class="technique-body">
    <p>App 서버에서 애플리케이션 경로, 컨테이너 구성, 환경 설정 파일을 확인하는 단계다. 단순 명령 실행보다 어떤 경로에 접근했는지와 그 접근 주체를 함께 본다.</p>
    <div class="mapping-grid">
      <div>
        <h3>Log IDs</h3>
        <p><a href="../logs/LL-002.html">LL-002 Linux auditd File Access Log</a></p>
      </div>
      <div>
        <h3>Representative Path</h3>
        <p><code>/var/log/audit/audit.log</code></p>
      </div>
      <div>
        <h3>Fields to Read</h3>
        <p><code>auditd.log.name</code>, <code>process.executable</code>, <code>user.name</code>, <code>auditd.log.key</code></p>
      </div>
    </div>
    <h3>Detection Point</h3>
    <ul>
      <li><code>/opt/spacebar-booking</code> 같은 애플리케이션 운영 경로 접근 확인</li>
      <li><code>/usr/bin/ls</code>, <code>cat</code>, <code>find</code> 등 경로 탐색성 프로세스 확인</li>
      <li>SSH 접속 직후 같은 세션에서 경로 접근이 발생했는지 확인</li>
    </ul>
    <h3>KQL Draft</h3>
    <pre><code>event.module: "auditd" and auditd.log.name: /opt/spacebar-booking*</code></pre>
    <h3>Investigation Pivot</h3>
    <p><code>auditd.log.key</code> -> <code>process.pid</code> -> <code>user.name</code> -> 직전 SSH 로그인 이벤트로 연결한다.</p>
  </div>
</details>

<details class="technique-panel">
  <summary><span class="id-cell">T1213.006</span> Data from Information Repositories: Databases</summary>
  <div class="technique-body">
    <p>App 서버에서 확인한 DB 접속 정보를 이용해 PostgreSQL 고객/예약 데이터를 조회하는 단계다. DB 로그는 쿼리 자체보다 접속 주체, 출발지, 대상 DB, 비정상 조회 패턴을 중심으로 본다.</p>
    <div class="mapping-grid">
      <div>
        <h3>Log IDs</h3>
        <p><a href="../logs/DBL-001.html">DBL-001 PostgreSQL Connection and Query Log</a></p>
      </div>
      <div>
        <h3>Representative Path</h3>
        <p><code>/var/log/postgresql/postgresql-*.log</code></p>
      </div>
      <div>
        <h3>Fields to Read</h3>
        <p><code>database.name</code>, <code>user.name</code>, <code>source.ip</code>, <code>event.action</code>, <code>query</code></p>
      </div>
    </div>
    <h3>Detection Point</h3>
    <ul>
      <li>App 서버가 아닌 위치에서 DB 접속이 발생했는지 확인</li>
      <li>예약/고객 테이블에 대한 대량 조회 또는 평소와 다른 시간대 조회 확인</li>
      <li>DB 접속 직전 App 서버에서 환경 설정 파일 접근이 있었는지 연결</li>
    </ul>
    <h3>KQL Draft</h3>
    <pre><code>service.type: "postgresql" and message: ("booking" or "reservation" or "customer")</code></pre>
    <h3>Investigation Pivot</h3>
    <p>DB 접속 계정 -> source.ip -> App 서버 auditd 이벤트 -> Jenkins/App SSH 이벤트 순으로 역추적한다.</p>
  </div>
</details>
