# FL-005 Falco IMDS Credential Access

| Field | Value |
| --- | --- |
| Log ID | FL-005 |
| Source | SB-04 K8s Node |
| Representative Path | `journalctl -u falco-modern-bpf.service` |
| Collection | Wazuh agent journald input (`/var/ossec/etc/ossec.conf`) |
| Primary Use | IMDS 접근을 통한 EC2 Instance Profile 임시 자격증명 획득 행위 확인 |

## 공격 행위 요약

FL-005는 NodeFall 캠페인에서 공격자가 탈취한 노드 또는 컨테이너 실행 환경에서 AWS EC2 Instance Metadata Service(IMDS)에 접근하여 Instance Profile 임시 자격증명을 획득하려는 런타임 행위를 설명하는 Falco 기반 로그이다.

공격자는 `169.254.169.254` 메타데이터 엔드포인트에 접근하여 IMDSv2 토큰을 발급받고, `/latest/meta-data/iam/security-credentials/` 경로를 조회하여 EC2 인스턴스에 연결된 IAM Role 이름과 임시 자격증명을 확인했다. 이 자격증명은 이후 STS, Secrets Manager, RDS 등 AWS API 호출에 사용될 수 있다.

Falco는 eBPF/driver를 통해 Linux syscall 및 프로세스 실행 정보를 관찰한다. 프로세스 커맨드라인에 IMDS 주소 또는 메타데이터 자격증명 경로가 포함되거나, `169.254.169.254`로 네트워크 연결이 발생하면 Falco rule `FL-005 IMDS Credential Access`가 매칭되고 `[FL-005] IMDS Credential Access` 로그가 생성된다.

FL-005는 CloudTrail의 `GetCallerIdentity`, `ListSecrets`, `GetSecretValue`, `DescribeDBInstances` 이벤트 이전 단계에서, AWS API 호출에 사용될 수 있는 임시 자격증명을 런타임에서 획득하려 한 증거로 사용된다.

## 로그 발생 위치

| 구분 | 위치 | 설명 |
| --- | --- | --- |
| K8s Node | Linux syscall / process execution | IMDS 접근 명령 실행 또는 메타데이터 IP로 네트워크 연결 발생 |
| AWS IMDS | `169.254.169.254` | EC2 Instance Metadata Service 엔드포인트 |
| Falco | rule `FL-005 IMDS Credential Access` | IMDS 접근 행위가 조건과 매칭되면 탐지 이벤트 생성 |
| K8s Node | `falco-modern-bpf.service` | Falco가 생성한 탐지 이벤트가 systemd service 로그로 기록 |
| K8s Node | `journalctl -u falco-modern-bpf.service` | 분석자가 Falco 원본 탐지 로그를 확인하는 대표 위치 |

## 수집 방식

| 구분 | 방식 |
| --- | --- |
| Falco 탐지 기준 | 프로세스 커맨드라인에 `169.254.169.254`, `/latest/api/token`, `/latest/meta-data/iam/security-credentials`가 포함되는지 확인 |
| Falco 탐지 기준 | 프로세스가 IMDS IP(`169.254.169.254`)로 네트워크 연결을 시도하는지 확인 |
| Falco 탐지 메시지 | 조건 매칭 시 Falco `output` 값인 `[FL-005] IMDS Credential Access`가 journald 로그에 기록됨 |
| Linux syscall | `execve`, `execveat`, `connect`, `sendto`, `sendmsg` 이벤트가 eBPF/driver를 통해 Falco에 전달됨 |
| journald | Falco 탐지 이벤트가 `falco-modern-bpf.service`의 systemd journal에 저장됨 |
| Wazuh agent | K8s node의 `/var/ossec/etc/ossec.conf`에서 journald 로그를 수집 |
| Wazuh manager | 수집된 Falco 로그를 `nodefall_falco_rules.xml`의 `FL-005` 룰과 매칭 |
| Wazuh Dashboard | 매칭된 alert를 `[FL-005] IMDS Credential Access` 이벤트로 표시 |

## 로그 출력 명령어

```bash
sudo journalctl -u falco-modern-bpf.service --no-pager | grep -E "\[FL-005\] IMDS Credential Access"
```
